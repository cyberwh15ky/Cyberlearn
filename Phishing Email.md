# Phishing Email
攻擊流程 and 防禦機制

### 🎯 釣魚郵件攻擊鏈（Phishing Kill Chain）概覽

| 階段 | 攻擊者行為 | 防禦者可做的事 |
|------|-------------|----------------|
| 1️⃣ 偵察 | 收集目標資訊（如公司架構、員工名單） | 使用 LinkedIn 保護設定、限制公開資訊 |
| 2️⃣ 偽造 | 製作假冒郵件與釣魚網站 | 實施 SPF/DKIM/DMARC 驗證 |
| 3️⃣ 發送 | 利用 SMTP relay 或 botnet 發送郵件 | 郵件閘道過濾、沙箱分析 |
| 4️⃣ 誘導 | 引導使用者點擊連結或下載附件 | 使用者教育、模擬演練（如 Gophish） |
| 5️⃣ 收集 | 收集帳密或植入惡意程式 | 網頁防詐、端點防護、SIEM 監控 |

### 🧪 想做合法模擬測試？你可以這樣開始：

1. **使用 Gophish**：這是開源的釣魚模擬平台，適合企業訓練。
2. **建立測試環境**：使用虛擬機與內部網路，避免誤傷真實用戶。
3. **撰寫教育性郵件模板**：例如「公司帳號異常登入通知」。
4. **設計釣魚頁面**：模擬登入頁面，並啟用資料收集（僅限測試）。
5. **啟動活動並觀察結果**：分析誰點擊、誰輸入資料，進行教育。



# 合法、可控的釣魚郵件模擬演練流程
## Gophish 為核心工具，搭配虛擬環境與最佳實務，讓你能在不觸法、不誤傷的前提下，完整執行一次演練。

---

### 🧪 模擬釣魚郵件演練流程（合法紅隊版本）

#### 1️⃣ 環境準備
- **建立隔離測試環境**（建議使用內網或虛擬機）
- 安裝 [Gophish](https://github.com/gophish/gophish)（開源釣魚模擬平台）
- 準備 SMTP 測試帳號（可用 Mailtrap、SendGrid sandbox）

#### 2️⃣ 設計演練範圍
- **取得授權**：確保有書面授權（紅隊規範）
- **定義目標群組**：選定測試對象（如內部員工、測試帳號）
- **設定 KPI**：如開信率、點擊率、資料提交率

#### 3️⃣ 建立釣魚活動元件
| 元件 | 說明 |
|------|------|
| 郵件樣板 | 偽裝 IT 通知、HR 公告等 |
| 登入頁面 | 模擬常見服務（如 OWA、VPN 登入） |
| 寄件設定 | 使用合法 SMTP，設定 SPF/DKIM 測試域名 |
| 追蹤設定 | 啟用開信追蹤、點擊追蹤、資料提交紀錄 |

#### 4️⃣ 啟動演練
- 啟動 Gophish Campaign
- 觀察即時統計（開信、點擊、提交）
- 記錄所有事件與時間戳記

#### 5️⃣ 結果分析與報告
- 匯出 CSV 統計報表
- 分析高風險行為（如輸入帳密者）
- 撰寫報告並提出改善建議（如教育訓練）

#### 6️⃣ 教育與回饋
- 舉辦資安意識訓練
- 分享演練結果與常見釣魚手法
- 建立回報機制（如 Outlook 報釣魚按鈕）

---

### 🔧 工具推薦
- **Gophish**：釣魚模擬平台
- **Mailtrap / Mailhog**：SMTP 測試伺服器
- **urlscan.io / VirusTotal**：釣魚頁面分析
- **PhishTool / PhishInsight**：進階分析與報告

---

這樣的流程不僅合法，還能有效提升組織的資安韌性。


# 紅隊釣魚郵件模擬演練 SOP（標準作業程序）文件
## 以 Gophish 為核心工具，適用於企業內部訓練、資安演練或教育用途。

---

## 🧾 紅隊釣魚演練 SOP（Standard Operating Procedure）

### 1️⃣ 目的與範圍
- **目的**：模擬釣魚攻擊，測試組織對社交工程攻擊的防禦能力。
- **適用對象**：內部 IT 安全團隊、紅隊成員、資安教育訓練單位。
- **範圍**：僅限於授權範圍內的內部帳號與網段，並使用隔離環境。

---

### 2️⃣ 前置準備

#### 2.1 授權與規範
- 撰寫並簽署 **RoE（Rules of Engagement）**
- 明確定義演練時間、目標人員、允許與禁止行為

#### 2.2 環境建置
- 安裝 Gophish 至內部伺服器或 VM
- 設定 SMTP 測試帳號（建議使用 Mailtrap、SendGrid sandbox）
- 建立隔離測試網段與 DNS 模擬環境（如釣魚頁面用網域）

---

### 3️⃣ 活動設計

#### 3.1 郵件樣板設計
- 偽裝主題：如「VPN 密碼過期通知」、「HR 表單更新」
- 嵌入追蹤連結與釣魚頁面（使用 Gophish 網頁模組）

#### 3.2 釣魚頁面設計
- 模擬登入頁面（如 OWA、Google Workspace）
- 表單欄位：帳號、密碼（僅作為教育用途，不儲存）

#### 3.3 活動設定
- 建立 Gophish Campaign
- 設定寄件人、收件人群組、寄送時間
- 啟用追蹤功能（開信、點擊、輸入）

---

### 4️⃣ 執行與監控

#### 4.1 活動啟動
- 確認 SMTP 正常運作
- 啟動 Campaign 並即時監控統計數據

#### 4.2 事件記錄
- 記錄開信、點擊、輸入帳密等行為
- 匯出 CSV 報表備份

---

### 5️⃣ 結果分析與回報

#### 5.1 報告內容
- 活動摘要（時間、對象、樣板）
- 成功率統計（開信率、點擊率、輸入率）
- 高風險使用者行為分析

#### 5.2 改善建議
- 建議加強 MFA、郵件過濾、資安教育
- 提供識別釣魚郵件技巧與案例

---

### 6️⃣ 教育與後續行動

- 舉辦資安意識訓練課程
- 分享演練結果與常見釣魚手法
- 建立「報釣魚郵件」機制（如 Outlook 插件）

# 紅隊演練手冊
涵蓋從規劃、執行到報告的完整流程，適合用於企業內部模擬攻擊演練或教育訓練。

---

## 🧨 紅隊演練手冊（Red Team Exercise Playbook）

### 📌 目標與原則
- **目的**：模擬真實攻擊情境，測試組織的偵測、防禦與應變能力。
- **原則**：合法授權、最小干擾、可追蹤、可回溯。

---

### 🧱 一、前期準備

#### ✅ 1.1 取得授權
- 撰寫 **Rules of Engagement (RoE)** 文件
- 明確定義演練範圍、時間、目標與限制

#### ✅ 1.2 建立測試環境
- 使用隔離網段或虛擬機群組
- 建立釣魚郵件測試帳號與 SMTP 測試伺服器（如 Mailtrap）

#### ✅ 1.3 工具準備
| 工具 | 用途 |
|------|------|
| Gophish | 釣魚郵件模擬平台 |
| Wireshark | 封包分析 |
| urlscan.io / VirusTotal | 網頁與附件分析 |
| Metasploit / Cobalt Strike | 攻擊模擬（需授權） |

---

### 🎯 二、演練設計

#### 🧩 2.1 攻擊鏈設計（可選擇以下模組）
- **釣魚郵件模擬**：偽裝 IT 通知、HR 公告
- **社交工程**：電話誘導、QR code 誘導
- **漏洞利用**：針對已知 CVE 進行滲透
- **橫向移動**：模擬內部網路橫向攻擊
- **資料外洩模擬**：模擬 exfiltration 行為

#### 🧠 2.2 偵測與回應測試
- 是否觸發 SIEM 告警？
- EDR 是否攔截？
- 藍隊是否回應？

---

### 🚀 三、演練執行

#### 📤 3.1 發送釣魚郵件
- 使用 Gophish 建立活動
- 啟用追蹤功能（開信、點擊、輸入）

#### 🕵️ 3.2 攻擊模擬
- 利用釣魚頁面收集帳密
- 模擬登入內部系統
- 嘗試提權與資料存取

#### 📈 3.3 即時監控
- 記錄所有事件與時間戳記
- 使用 ELK / Splunk 觀察藍隊反應

---

### 📊 四、結果分析與報告

#### 📋 4.1 報告內容
- 攻擊路徑圖（Kill Chain）
- 成功與失敗的攻擊手法
- 偵測與回應時間
- 改善建議（技術與流程）

#### 🧾 4.2 報告格式建議
- Executive Summary（給管理層）
- Technical Findings（給資安團隊）
- 改善建議與優先順序

---

### 🧠 五、教育與強化

- 舉辦資安意識訓練
- 分享釣魚郵件範例與識別技巧
- 建立報釣魚機制（如 Outlook 插件）
